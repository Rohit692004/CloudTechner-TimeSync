Innovation: Smart Comments Validation

Dynamic Compliance Rules

  • Project-Specific Configurations: Settings for "No Comments", "Mandatory", or "Log-time Based".
  • The Deviation Standard: Comments are required only if the log is less than or greater than 8 hours (x < 8 < x).
  • Standard Logs: Clean 8-hour logs bypass validation, speeding up submission rates.

Automated DB Cleaning

  • Server-Side Actions: Auto-prunes note fields if an employee changes a non-standard hour entry back to 8 hours.
  • Benefits:
    • Prevents orphan records.
    • Ensures audit tables only hold valid explanations.
    • Reduces database storage clutter.

Onboarding & Allocation Pipeline

Self-Service Requests

  • Employee Portal: Unassigned staff raise allocation requests directly in-app.
  • Metadata Input: Specifies target project, start/end dates, allocation percentage, and message.

HR / Admin Controls
